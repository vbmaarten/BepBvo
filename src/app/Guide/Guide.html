Introduction

Welcome to Code Stories. Code Stories was written as a Bachelors project at the TU Delft. Code Stories allows a developer to guide users through a project and visualize code. We only support a JS interpreter at the moment.

Basic usage

Viewer

To view a narrative select 'viewer mode' ( the eye at the top right) and then select a narrative. You can auto play it with the play button. The button allows you to step to the next item and the will allow you to do a single step when viewing a code narrative

Writer

There are two kinds of narratives. Code Narratives and FileSystem Narratives.

FileSystem narratives can be placed on a file or folder node in a project In this case they are a simple sequence of items that will be displayed. You can also add link items inside a FS Narrative that will push that narrative on the stack. once that narrative is done it will continue where you left off. Code Narratives are a little different. You click in the code view to select an AST Node. This is the scope of the item hooks you can place. Now you can place items at points inside the code you have selected. Every time this sub AST Node is popped from the stack, the item will be displayed to the viewer. There are three ways you can interact with the code when you are writing a code narrative.

You can insert variables in your text items by using [[ variable_name ]] .

You can execute some code inside the interpreter with the code item. This is most usefull if you want to initialize your function. Placing a code item on the '/' node will execute after the code has been initiated.

You can run 'VCode' (Visualization items) . VCode has access to the variables in the scope. And it holds a persistant scope of its own. ( variables declared in one VCode item can be used in any other VCode item within the same CodeNarrative )

As a writter you can create visualization objects to be called from your narrative. These are wrappers to keep down the size and complexity of your story items. You can edit and them in the Visualization tab.

Known Issues

VCode is evaled , more info in the source code ('VCode/vcodeinterpreter.factory.js') The narrator is fixed width and VCode items are not dynamic in size.
